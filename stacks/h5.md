# Stack Pack: h5 / 移动 Web 前端

## 1. 默认技术栈 / Default Stack
- Framework: React / Vue 3 + Vite + TypeScript
- 样式: Tailwind / CSS Modules；移动适配 viewport + rem/vw
- 关键: 路由(react-router/vue-router)、请求封装、首屏性能

## 2. 目录布局约定 / Layout Conventions
```
src/
├── pages/          # 路由页面
├── components/      # 复用组件
├── router/         # 路由表
├── api/            # 接口封装
├── store/          # 状态
└── styles/
```

## 3. Boundary 拆分模式 / Parallelization Patterns ⭐
- 天然可并行：**按页面/路由切**、按独立组件切、按 api 模块切——各自文件互不重叠。
- 易冲突共享文件：`router` 路由表、`store` 入口、全局样式/主题、`main.ts`。
- 推荐 Boundary：一个任务 = 一个 page 或组件 + 其样式；路由/store 注册放收尾串行任务。

## 4. 测试策略 / Testing
- 组件测试(Testing Library) + 关键逻辑单测；视觉/交互可用 Playwright e2e。
- 浏览器行为可在 verify 阶段用 Preview/Playwright 实跑核对。

## 5. 领域红线与常见坑 / Red Lines & Pitfalls
- ❌ 把密钥/敏感逻辑放前端；❌ 阻塞首屏的大依赖未拆分。
- ⚠️ 移动端 1px/安全区/软键盘、iOS Safari 兼容、首屏与 LCP、图片懒加载、XSS、跨域。

## 6. 验收要点 / Acceptance Focus
- 移动端多机型适配、首屏性能(LCP/FCP)、可访问性、弱网降级、表单校验与无障碍。

## 7. 本层质量门禁 / Layer Quality Gate
- Format: `prettier --write 'apps/web/**/*.{ts,tsx,vue,css}'`
- Lint / Typecheck: `eslint apps/web` · `vue-tsc --noEmit`（或 `tsc --noEmit`）
- Test: `pnpm --filter web test`（Vitest / Testing Library）
